# VideoDownloader — project guide

Native **macOS SwiftUI** app: paste a video URL (or playlist), pick a format, download it. A curated GUI over **yt-dlp + ffmpeg**. Personal use, macOS 14+, non-sandboxed.

## Layout

Two build units:

- **`VideoDownloaderCore`** — a dependency-free SwiftPM library (`Sources/VideoDownloaderCore/`) holding all the pure logic (models, yt-dlp JSON parsing, argument building, progress parsing, binary management, download engine, queue state machine). Fully unit-tested with XCTest.
- **App shell** — a thin SwiftUI layer under `App/`. The Xcode project is **generated by XcodeGen** from `App/project.yml` (the generated `App/VideoDownloader.xcodeproj` is gitignored — never commit it).

Design & implementation docs live in `docs/superpowers/specs/` and `docs/superpowers/plans/`.

## Build / test / run

```bash
# Library (fast TDD loop — no GUI needed)
swift test          # runs the whole XCTest suite
swift build

# App (requires: brew install xcodegen)
cd App && xcodegen generate
xcodebuild -project App/VideoDownloader.xcodeproj -scheme VideoDownloader \
  -configuration Debug -destination 'platform=macOS' build   # expect: ** BUILD SUCCEEDED **

# Run it: open the generated project and press ⌘R
open App/VideoDownloader.xcodeproj
```

Both gates (`swift test` green **and** `xcodebuild` BUILD SUCCEEDED) should stay green for any change.

## Runtime notes / gotchas

- **Binaries are downloaded at first launch**, not bundled: yt-dlp + an arch-matched static ffmpeg/ffprobe land in `~/Library/Application Support/VideoDownloader/bin/`. Delete that folder to re-test the first-run setup.
- The app is **ad-hoc signed** (`CODE_SIGN_IDENTITY: "-"`, no Apple team). Downloaded binaries are unquarantined + ad-hoc signed (arm64) so Gatekeeper lets them run.
- **`UNUserNotificationCenter` can hang for dev/unsigned builds launched from Xcode** (you'll see `linkd` / intents registration errors in the console). Notification permission is therefore requested *off the critical path* (fire-and-forget in `AppModel.bootstrap`) so it can never block startup. Those console warnings (ViewBridge, CoreAudio LoudnessManager, DetachedSignatures, etc.) are benign macOS system noise, not app bugs.
- The app icon is generated by `scripts/make-appicon.swift` (`swift scripts/make-appicon.swift`) into `App/Assets.xcassets/AppIcon.appiconset/`.

## Out of scope (v1)

Subtitles, login/cookies for private content, editing/trim, distribution/notarization, queue persistence across launches, Windows/Linux.
